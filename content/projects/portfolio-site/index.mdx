## 개인 포트폴리오 사이트

Next.js 14와 Tailwind CSS를 활용하여 구축한 개인 포트폴리오 웹사이트입니다. MDX 기반의 콘텐츠 관리 시스템으로 프로젝트를 손쉽게 관리할 수 있습니다.

### 프로젝트 개요

프론트엔드 개발자로서의 작업물을 효과적으로 전시하고 소개하기 위한 포트폴리오 사이트입니다. 사용자 경험을 최우선으로 고려하여 설계했습니다.

### 주요 기능

- **MDX 기반 콘텐츠**: 마크다운으로 프로젝트 콘텐츠를 작성하고 관리
- **반응형 디자인**: 모든 디바이스에서 최적화된 사용자 경험
- **다크 테마**: 눈의 피로를 줄이는 다크 모드 인터페이스
- **타이포그래피**: 프리텐다드 폰트로 한글 가독성 최적화
- **빠른 로딩**: Next.js의 정적 생성으로 초고속 페이지 로딩

### 기술 스택

#### 프론트엔드
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- MDX (next-mdx-remote)

#### 디자인
- Pretendard 폰트
- 다크 테마 디자인
- 반응형 레이아웃
- Hero 섹션 디자인

### 개발 하이라이트

#### 1. 타이포그래피 최적화
웹 타이포그래피 가이드라인을 기반으로 한글 콘텐츠에 최적화된 자간, 행간, 글자 크기를 설정했습니다.

- 본문: 16px, 행간 1.75
- 제목: 자간 -0.02em, 행간 1.3
- 프리텐다드 폰트 적용

#### 2. MDX 기반 콘텐츠 시스템
마크다운으로 프로젝트 내용을 작성하고 자동으로 웹 페이지로 변환되는 시스템을 구축했습니다.

- Frontmatter로 메타데이터 관리
- 자동 URL 생성
- 동적 라우팅

#### 3. 성능 최적화
Next.js의 정적 생성(SSG)과 이미지 최적화를 활용하여 빠른 로딩 속도를 달성했습니다.

- Static Site Generation
- 자동 이미지 최적화
- 코드 스플리팅

### 디자인 컨셉

#### 미니멀리즘
불필요한 요소를 제거하고 콘텐츠에 집중할 수 있도록 깔끔한 디자인을 추구했습니다.

#### 타이포그래피 중심
아름다운 타이포그래피를 통해 전문성을 표현하고 가독성을 극대화했습니다.

#### 다크 모드
현대적인 다크 테마로 눈의 피로를 줄이고 세련된 느낌을 전달합니다.

### 개발 과정

#### 초기 설정
Next.js 14 프로젝트를 생성하고 Tailwind CSS를 설정했습니다. TypeScript를 도입하여 타입 안정성을 확보했습니다.

#### 콘텐츠 시스템 구축
MDX 파일을 읽고 파싱하여 웹 페이지로 렌더링하는 시스템을 구현했습니다. 프로젝트 추가가 매우 간편해졌습니다.

#### 디자인 개선
여러 차례의 반복을 통해 타이포그래피, 색상 대비, 레이아웃을 개선했습니다. 사용자 피드백을 반영하여 가독성을 향상시켰습니다.

### 배운 점

이 프로젝트를 통해 Next.js 14의 새로운 App Router에 대해 깊이 이해하게 되었습니다. 특히 정적 생성과 동적 라우팅을 효과적으로 조합하는 방법을 배웠습니다.

또한 웹 타이포그래피의 중요성을 체감했습니다. 적절한 글꼴 선택, 자간, 행간 설정이 전체적인 사용자 경험에 큰 영향을 미친다는 것을 알게 되었습니다.
